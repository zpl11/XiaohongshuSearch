# Skill: 小红书微型工具需求监控

## 使命
专注于在小红书平台上发现具有“微型 SaaS”或“微型工具”潜力的用户痛点和搜索趋势。

## 方法论
1. **场景与职业关键词生成**: 放弃直接搜索“工具”，转而通过 AI 生成“职业 + 负面情绪/高频场景”的组合词。
   - **职业类**: 会计、新媒体运营、外贸员、电商客服、HR。
   - **情绪/场景类**: 熬夜、手动、重复、整理、要命、崩溃、有没有简单方法。
   - **示例组合**: “财务 报销 吐槽”、“运营 找素材 痛苦”、“外贸 发盘 整理 手动”。
2. **内容定位**: 检索小红书上高互动的“职场吐槽类”、“避雷类”或“效率低下的生活/工作片段”笔记。
3. **评论深度挖掘**: 重点分析上述笔记的评论区，寻找类似“要是有个能...的东西就好了”或“我每天都在花 3 小时做这个”这种真实的、隐藏的原始痛点。

## 进化策略
- **高势能职业定位**: 如果某个职业（如“跨境电商”）下的吐槽内容频繁出现可自动化的操作点，则自动锁定该职业。
- **痛点模式库更新**: 将“重复性最高”且“现有成熟 SaaS 无法覆盖”的小切口场景标记为高变现优先级。

## 输入
- 基础关键词库（微型 SaaS/工具类）。
- 小红书 API 或爬取数据流（模拟）。

## 输出
- 初步筛选的“潜在需求笔记列表”。
- 对应的搜索关键词热度评分。
